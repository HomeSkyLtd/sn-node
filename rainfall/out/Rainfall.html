Message

The message object. It is a regular javascript object with certain fields. Depending on the packageType (which is obligatory) and on nodeClass (when present). To define enums you can use the enum item or a string with the key. Flaggable enums accept multiple values (like packageType and nodeClass)
Type:
  • Object
Properties:
Name Type Attributes Description
packageType PACKAGE_TYPES | String Defines which package it is. Required field and the presence or absence of other fields depends on this value.
nodeClass NODE_CLASSES | String <optional>
 Field to define which class the node is
yourId Number <optional>
 Used by the controller to inform the node its id.
id Number <optional>
 Field defining the node id. It is useful to the controller to know which node is sending the message. Defined by the controller in the yourId field.
data Array.<Rainfall~data> <optional>
 List of data. Sent by the sensor to the controller.
command Array.<Rainfall~data> <optional>
 List of commands. Sent by the controller to the actuator.
lifetime Number <optional>
 The period of time between keepalive messages sent by the node
dataType Array.<Rainfall~dataType> <optional>
 Field used by the sensor to inform the controller the data it will send
commandType Array.<Rainfall~commandType> <optional>
 Field used by the actautor to inform the controller the commands it accepts
